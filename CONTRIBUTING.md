# Contributing

Keep the skill sharp.

Good contributions usually do one of three things:

- make the trigger more precise
- make the judgment framework easier to apply
- add a better example

Avoid:

- impersonation
- long theory sections
- extra process
- more words when deletion would do

Before opening a PR:

```bash
npx skills add . --list
claude plugin validate .
```

The canonical skill is:

```text
skills/rick-rubin/SKILL.md
```
