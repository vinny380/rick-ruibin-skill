# Examples

The skill works best when the artifact is real.

Not a premise.

Not a pitch.

The thing.

## Product Scope

```text
Input:
"AI dashboard with insights, tasks, team updates, automations, integrations,
and weekly strategic recommendations."

Verdict: overbuilt. It sounds useful, but I already want to close it.

Essence: "tell me what changed and what needs me."

Cut: dashboards, weekly strategy, vague insights, generic automations.

One change: make the first version a daily operator brief:
5 changes, 3 risks, 1 thing to do now.
```

## Landing Copy

```text
Input:
"Supercharge your workflow with next-generation AI-powered productivity
for modern teams."

Verdict: flat. Every word is polished. None of them are alive.

Essence: the product probably saves someone from a specific daily irritation.

Cut: supercharge, next-generation, AI-powered, modern teams.

One change: name the painful moment:
"Know what changed before your first meeting."
```

## Code Design

```text
Input:
"Should we add a StrategyFactoryProviderRegistry for choosing the parser?"

Verdict: no. That abstraction is wearing a suit to buy milk.

Essence: choose one parser from a small known set.

Cut: factory, provider, registry.

One change: use a plain map until the complexity proves it exists.
```
